# Maven

Maven概念：

是一个纯 Java开发的开源项目。基于项目对象模型（缩写：POM）

Maven利用一个中央信息片断能管理一个项目的构建、报告和文档等步骤。

Maven 是一个项目管理工具，可以对 Java 项目进行构建、依赖管理。

Manven生命周期

默认生命周期：compile,test, package,install,deploy

核心功能：依赖管理、一键构建

常用的命令：

Mvn clean: 清理项目生产的临时文件，一般是模块下的target目录

Mvn compile: 编译主程序

Mvn test-compile：编译测试程序

Mvn test: 执行测试 src/test/java/下junit的测试用例

Mvn package：项目打包，会在模块下的target目录生成jar 或war 文件

Mvn install: 模块安装命令，将打包的jar/war 文件复制到你的本地仓库中，供其他模块使用

Dmaven.test.skip=true 跳过测试和跳过test compile

Mvn deploy: 发布命令，将打包的文件放置私服，供其他人使用下载依赖

Idea 中配置Maven

idea中配置Maven 优先从本地仓库中获取

VM Option选项加上 参数：-DarchetypeCatalog=internal 加快项目构建速度

Maven 管理的 Pom.xml 完整的web项目目录结构,这里有out目录是因为没设置项目的编译路径
